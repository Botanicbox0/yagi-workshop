이 가이드는 `/app/admin/challenges/new` 페이지에서 새 챌린지를 등록할 때 각 필드가 어떤 의미를 갖는지 한눈에 보기 위해 만든 문서입니다. 내용은 MVP 초기 문서이며, 실제 운영 케이스가 쌓이면서 계속 보강됩니다.

## 1. 제목과 설명 — 참여 결정의 80%

챌린지 카드에 가장 먼저 노출되는 두 줄입니다. 제목은 "무엇을 만들 것인가"를, 설명은 "왜 만들 가치가 있는가"를 명확히 담아 주세요. 제목은 60자 이내, 설명은 줄바꿈 없이 3-4문장 안에서 정리하면 공모전 리스트 페이지에서 깨지지 않습니다. 크리에이터는 제목과 설명만으로 5-10초 안에 참여 여부를 결정하므로, 모호한 카피보다 구체적 동사가 항상 유리합니다. (예: "AI로 만드는 브랜드 필름" → "카페 브랜드의 15초 모션 포스터 제작")

## 2. 제출 요건 — 어떤 포맷을 받을 것인가

`submission_requirements` 체크박스는 제출물 형식을 강제합니다. 각 항목은 독립적으로 동작하므로 필요한 것만 켜세요.

- **영상 포함 (`video`)**: 제출물에 최소 1개의 비디오 파일이 반드시 포함돼야 통과합니다. 정지 이미지 챌린지에서는 끄세요.
- **이미지 포함 (`image`)**: 최소 1개의 이미지 파일 필수. 영상 전용 챌린지에서 이미지까지 요구하면 제출 허들이 올라갑니다.
- **설명 텍스트 (`description`)**: 크리에이터가 제출물에 자신의 제작 의도를 짧게 적어야 합니다. 심사 모드가 공개 투표일 때 특히 중요합니다.
- **소스 자산 (`source_assets`)**: 원본 프로젝트 파일(AE, 블렌더, 래스터 PSD 등) 제출을 강제합니다. 수상 후 활용을 전제로 하는 챌린지에서만 켜세요 — 평소에는 과도한 요구가 됩니다.

## 3. 심사 모드 — 누가 결정할 것인가

세 가지 중 하나를 고릅니다.

- **관리자 심사 (`admin_only`)**: YAGI 내부 심사단이 비공개로 선정합니다. 브랜드 의뢰 프로젝트의 선발, 합작 파트너 선정 등 "결과가 사업적으로 뒷단에 붙는" 챌린지에 맞습니다.
- **공개 투표 (`public_vote`)**: 로그인한 커뮤니티 구성원이 각 제출물에 1표씩 응원합니다. 브랜딩 공모전, 팬 아트 기반 이벤트에 적합합니다. 참여 수 자체가 목적일 때 선택하세요.
- **하이브리드 (`hybrid`)**: 투표 상위 N개 중에서 관리자가 최종 수상자를 고릅니다. "대중성과 완성도를 둘 다 보고 싶을 때"의 기본값입니다.

## 4. 타임라인 — 언제 열고, 언제 닫고, 언제 발표할까

세 개의 시각을 모두 정확한 현지 시간(KST 기준)으로 입력하세요.

- **`open_at`**: 공개 시점. 이 시각 이전에는 챌린지가 목록에 노출되지 않고, 제출도 불가합니다.
- **`close_at`**: 제출 마감. 일반적으로 공개 후 2-4주를 권장합니다. 너무 짧으면 실작업 크리에이터가 참여할 수 없고, 너무 길면 모멘텀이 사라집니다.
- **`announce_at`**: 결과 발표 시점. 공개 투표 모드에서는 마감 후 최소 48시간 이상 간격을 둬서 집계/검토 시간을 확보하세요. 관리자 심사 모드에서는 7-10일의 심사 공백이 자연스럽습니다.

## 발행 전 체크리스트

- [ ] 제목·설명은 리스트 페이지에서 잘리지 않는가
- [ ] 제출 요건이 최소한만 켜져 있는가 (기본값: 영상 또는 이미지 + 설명)
- [ ] 심사 모드가 이 챌린지의 성격에 맞는가
- [ ] `open_at < close_at < announce_at`이 시간순으로 성립하는가
- [ ] 상금/혜택/수상자 수가 설명에 명시돼 있는가

이 다섯 가지를 확인하면 "발행했는데 아무도 참여하지 않는" 최악의 케이스는 거의 방지됩니다. 운영 중 발견한 새 엣지 케이스는 이 문서로 계속 피드백해 주세요.
