## Notes

- `validateVideoFile` enforces MIME in {video/mp4, video/quicktime, video/webm}
  and size <= 500MB. `readVideoMetadata` uses a hidden, off-DOM `<video>` +
  `<canvas>` (max 640px wide) to capture a JPEG poster at t=1.0s (clamped to
  `duration - 0.1s` if shorter). Silent on every failure mode — returns
  `{ duration_seconds: null, poster: null }`. Wrapped in a 15s safety timeout.

- `unfurlVideoUrl` inlines the Phase 1.2 SSRF guard (IPv4/IPv6 private-range
  check + DNS resolution) from `src/lib/og-unfurl.ts` since that helper isn't
  exported. Hostname dispatch:
    - YouTube (youtube.com, m.youtube.com, youtu.be, music.youtube.com,
      all with/without www) -> oembed
    - Vimeo (vimeo.com, player.vimeo.com) -> oembed, reads `duration`
    - TikTok (tiktok.com, vm.tiktok.com, m.tiktok.com) -> oembed
    - Instagram (instagram.com, instagr.am) -> graph.facebook.com
      `instagram_oembed`, requires `INSTAGRAM_OEMBED_TOKEN`; returns null
      when token is absent so caller falls back to generic OG unfurl.
  Non-OK status, non-JSON content-type, missing title/thumbnail_url, unknown
  hostname, or any exception all map to `null`. 8s timeout per call; response
  body cap 200KB. Never throws.

- `addReference` schema extended with optional `media_type` (enum image|video|pdf),
  `duration_seconds` (non-negative finite), `thumbnail_path`, `embed_provider`
  (enum youtube|vimeo|tiktok|instagram), `page_count`. Default logic:
  `media_type` defaults to "video" when `embed_provider` is set, otherwise
  "image". Path-safety guard: `storage_path` and `thumbnail_path`, if
  provided, must start with `{projectId}/` or the action returns
  `{ error: "validation" }`.

- `addReferenceFromUrl` server action: calls `unfurlVideoUrl` first; on video
  hit inserts with `media_type='video'`, `embed_provider`, `external_url=canonical`,
  `og_title`, `og_image_url`, `duration_seconds`. On null falls back to the
  generic `unfurl()` from `src/lib/og-unfurl.ts` (Phase 1.2) and inserts with
  `media_type='image'`, `embed_provider=null`. RLS enforces membership.

- `removeReference` updated to also delete `thumbnail_path` from storage when
  present, since video references now have companion poster objects.

- `reference-uploader.tsx` extended (not rewritten): new `<TabsTrigger
  value="video">` between image and url; dropzone `accept` map now includes
  `video/mp4`, `video/quicktime`, `video/webm`, `application/pdf`; dropzone
  `maxSize` bumped to 500MB with per-type manual validation in `onDrop`
  (image 10MB, video 500MB, PDF 25MB). Video branch runs
  `validateVideoFile` then `readVideoMetadata`, uploads the file, uploads
  the poster blob (if any) at `{projectId}/{uuid}_poster.jpg`, then calls
  `addReference` with `media_type='video'`, `duration_seconds`, and
  `thumbnail_path`. PDF branch uploads with `media_type='pdf'` (subtask 05
  will layer real handling). URL tab now calls `addReferenceFromUrl`
  directly instead of the old /api/unfurl + addReference duo; helper text
  below the input uses `t('ref_video_url_ph')`.

- All user-facing strings come from `useTranslations('refs')` and the
  existing `errors` namespace; no hardcoded strings introduced. No new
  dependencies added.
