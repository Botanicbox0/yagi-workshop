## key implementation details

### date+time combination
`new Date(`${date}T${time}:00+09:00`).toISOString()` — the browser-native Date constructor accepts the `+09:00` offset literal, producing a correct UTC ISO string for the `scheduledAt` field.

### member checkbox merge
1. The new-meeting page fetches workspace members using `profiles!workspace_members_user_id_fkey` to resolve the FK ambiguity (workspace_members has two FKs to profiles: user_id and invited_by).
2. Email is pulled from workspace_invitations (accepted only) since `auth.users.email` is not accessible via RLS. For the current user, `user.email` is used directly from the session.
3. The form shows workspace members as checkboxes. Checking a member appends them to the `useFieldArray`. Their row is rendered read-only (email field). Unchecking removes them from the array and updates the `checkedMemberEmails` Set.
4. Custom attendees are appended via the "Add attendee" button and have editable email fields.
5. On submit the union of checked members + custom rows is submitted as `attendeeEmails` / `attendeeDisplayNames` arrays.

### durationMinutes schema
`z.enum(["30","45","60","90"])` without `.transform(Number)` to avoid the `z.infer` / `zodResolver` type conflict. `Number(data.durationMinutes)` is called at submit time and cast as `30 | 45 | 60 | 90`.

### router.push for detail page
Uses `/${locale}/app/meetings/${result.meetingId}` with a type assertion since the detail route does not exist yet (subtask 09 will create it).

## upstream issues discovered
- `workspace_members` has two FK relationships to `profiles` (user_id and invited_by), requiring the explicit Supabase hint `profiles!workspace_members_user_id_fkey` in the select query. Without it TypeScript infers a `SelectQueryError` type.
- Member emails are not available via RLS-safe Supabase queries (`auth.users` is inaccessible). Workaround: use `workspace_invitations` accepted rows for email, and `user.email` from the session for the current user. Members who joined without an invitation will show an empty email and won't appear in the checkbox list (filtered out in the form).
