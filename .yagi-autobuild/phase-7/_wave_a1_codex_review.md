## VERDICT: NEEDS-ATTENTION

[FINDING 1] HIGH-B: `supabase/migrations/20260506000000_phase_7_campaigns.sql:384` — sponsor `INSERT` is row-gated but not column-gated. Because only `UPDATE` is column-locked at `441-443`, a malicious sponsor can create a `status='requested'` row while seeding admin/audit fields like `decision_metadata`, `has_external_sponsor`, `external_sponsor_name`, `compensation_*`, `allow_*`, dates, or forged `created_by`. Recommended fix: `REVOKE INSERT ON campaigns FROM authenticated`, grant only sponsor-request insert columns, and add `created_by = auth.uid()` to the sponsor `WITH CHECK`.

[FINDING 2] MED-A: `supabase/migrations/20260506000000_phase_7_campaigns.sql:679` — applicant `campaign_distributions` insert is correctly parent/status/workspace-gated, but insert columns are unrestricted. Applicant can forge `added_by`, `created_at`, `updated_at`, and possibly seed `notes` while creating an otherwise valid distribution. Recommended fix: column-lock `INSERT` for authenticated, or force `added_by = auth.uid()` / system timestamps via trigger and deny applicant-supplied audit/admin columns.

[FINDING 3] MED-A: `supabase/migrations/20260506000000_phase_7_campaigns.sql:744` — DO-block grant assertions are incomplete. UPDATE assertions miss `id`/timestamp or FK columns across tables: `campaigns.id`, `campaigns.created_at`, `campaign_categories.id`, `campaign_categories.campaign_id`, `campaign_submissions.id`, `campaign_submissions.created_at`, `campaign_review_decisions.id`, `campaign_review_decisions.decided_at`, `campaign_distributions.id`, `campaign_distributions.created_at`. It also does not assert the INSERT column matrix, which would have caught Findings 1-2.

Confirmed passes: sponsor workspace kind gate excludes future `creator`; applicant self-promotion to `approved_for_distribution` is blocked by `status IN ('withdrawn','distributed')`; `submitted -> distributed` is blocked by the OLD-aware trigger; `campaign_distributions` insert requires own approved parent submission; public showcase scope is status-based as intended; `workspace_members(workspace_id,user_id)` is covered by the unique constraint; `campaign_submissions.category_id` index is already present.

One-line summary: RLS shape is mostly solid, but authenticated INSERT column lockdown is missing on the two direct client insert paths, so A.1 should be patched before `mcp.apply_migration`.
